Hybrid Vectorial Similarity & Deterministic Rule Engine
🧮
Layer 1 — FAISS Vector Search
Projects invoice features into a multidimensional space using L2 distance on GPU. Identifies "near-duplicate" neighbors that standard filters miss.
Index: faiss.IndexFlatL2
Device: GPU (index_cpu_to_gpu)
Algorithm: k-nearest neighbors
Threshold: ≥ 95% similarity
Features: CoCode · Vendor · Ref · Date · Amount
Encoding: Numeric + StandardScaler
🎯
Layer 2 — Exact Match Engine
Relational join engine using pd.merge for 1:1 identity across business-critical keys. Deterministic fallback ensuring no true duplicate is missed.
Method: pd.merge (inner join)
Keys: CoCodes_duplicates
     + Vendor_code_duplicates
     + Reference_doc_duplicates
Output: Exact pairs (any date/amount)
Priority: High (always included)
🔤
Layer 3 — Fuzzy Levenshtein
String-distance refinement that calculates edit distance on Reference_doc to capture typos and character variations (e.g. "EX-123" vs "EX123").
Library: Levenshtein
Metric: Normalized edit distance
Threshold: ≤ 0.1 (≥ 90% similar)
Trigger: CoCode + Vendor + Date + Amt exact
Output: Case 2 classification
Vectorized: np.select operations
Duplicate Case Classifications
1
Exact Duplicate
CoCode + Vendor + Reference + Date + Amount — all five fields match exactly
2
Fuzzy Reference
CoCode + Vendor + Date + Amount exact, Reference doc fuzzy match (Levenshtein ≤ 0.1)
3
Partial Match
CoCode + Vendor + Reference + Date match — Amount may differ
4
Broader Match
CoCode + Vendor + Reference match only — broadest duplicate signal
